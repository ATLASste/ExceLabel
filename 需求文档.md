# 项目需求文档 (PRD)：ExcelLabel 工具

## 1. 项目概述
开发一个高性能的本地文件管理工具，能够将指定文件夹下的所有文件元数据提取并同步至 Excel 表格中。用户在 Excel 中修改文件名后，本地文件系统应实时或批量同步更改；反之，本地文件系统的变动也应反映在 Excel 中。

## 2. 核心功能需求

### 2.1 文件扫描与数据导出 (核心输出格式)
工具需递归扫描目标文件夹，并将信息按以下格式写入 Excel 单元格：

| 列序号 | 字段名称 | 说明 |
| :---: | :--- | :--- |
| **A列** | **文件名** | 不含后缀的文件原始名称 |
| **B列** | **文件扩展名** | 文件的后缀名（如 `.xlsx`, `.pdf`, `.dwg`） |
| **C列** | **子文件夹名** | 文件相对于根目录的路径（若在根目录则为空） |

### 2.2 双向同步机制 (Bi-directional Sync)
- **Excel -> 文件系统**：监听 Excel 单元格（A列）的编辑事件。当用户修改 A 列内容并保存或失焦时，程序自动执行 `rename` 操作。
- **文件系统 -> Excel**：监听目标文件目录。若用户在外部手动重命名、移动或删除文件，Excel 中的对应行应自动即时更新或标记。

### 2.3 智能处理逻辑
- **冲突检测**：重命名之前需检查目标文件名是否已在目标路径中存在，若存在需高亮提醒用户以防止覆盖。
- **非法字符过滤**：自动拦截或替换 Windows 文件系统不支持的字符（如 `\ / : * ? " < > |`）。
- **性能要求**：要求能够支持处理数千个文件，并达到毫秒级的响应速度，确保避免前端界面卡顿。

## 3. 技术栈建议
- **方案**：使用专业桌面端工具架构技术栈
  - **核心后端逻辑**：Go 
  - **前端GUI/框架**：Wails (Go) 
  - **Excel 表格处理**：`excelize` (Go) 
  - **文件系统监听**：`fsnotify` (Go) 

## 4. UI/UX 要求
- **路径选择**：提供直观易用的本地文件夹选择器组件。
- **一键同步操作**：若不采用全自动实时同步，需提供明确的“执行重命名”和“刷新列表”操作按钮。
- **状态反馈**：在界面日志区域或 Excel 最后一列表格中，清晰显示单项操作状态（如“✅ 修改成功”、“❌ 错误：文件被占用”等）。
- **界面设计风格**：前端界面采取浅色系现代化云母 (Mica) / 亚克力 (Acrylic) 透视毛玻璃风格设计，打造高级视觉体验。
