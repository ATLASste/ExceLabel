# ExceLabel

ExceLabel 是一个面向 Windows 的本地桌面工具，用于递归扫描指定目录中的文件，将文件元数据导出到 Excel，并在 Excel 与本地文件系统之间进行双向同步。它适合用于批量整理文件名称、建立可编辑清单，以及在 Excel 中完成受控改名后回写到磁盘。

## 功能特性

- 递归扫描目标目录并生成 Excel 工作簿清单
- 以 `文件名`、`扩展名`、`相对目录`、`状态` 的形式展示文件记录
- 支持在 Excel 中编辑文件名并批量回写到文件系统
- 支持监听文件系统变化并回写到工作簿
- 内置冲突检测，避免同目录重名覆盖
- 内置 Windows 非法文件名字符校验
- 通过隐藏元数据表维持 Excel 行与真实文件的稳定映射
- 支持工作区状态持久化，便于后续刷新与同步
- 使用 Go 并发扫描，适配数千文件规模的本地处理场景

## 适用场景

- 批量整理项目资料、图纸、合同、归档文件
- 用 Excel 作为“文件命名台账”进行人工校对
- 在本地文件夹与 Excel 清单之间建立可追踪的同步流程
- 对文件改名过程需要状态反馈、冲突提醒、回滚前确认的办公场景

## 核心工作流

1. 选择需要管理的根目录
2. 新建工作簿或选择已有工作簿
3. 初始化工作区，导出当前文件快照到 Excel
4. 在 Excel 中修改 `NameStem` 列并保存工作簿
5. 由应用执行改名同步，将 Excel 修改应用到实际文件
6. 当文件系统发生变化时，刷新工作区并回写 Excel 状态

## 工作簿设计

ExceLabel 默认使用两个工作表：

### `Files`

面向用户可见，包含以下字段：

| 列 | 字段 | 是否可见 | 是否可编辑 | 说明 |
| --- | --- | --- | --- | --- |
| A | NameStem | 是 | 是 | 不含扩展名的文件名 |
| B | Ext | 是 | 否 | 文件扩展名 |
| C | RelativeDir | 是 | 否 | 相对根目录的子路径 |
| D | Status | 是 | 否 | 同步状态、冲突或错误说明 |

### `_meta`

隐藏工作表，用于保存以下元数据：

- `RecordID`
- `FileFingerprint`
- `LastKnownPath`
- `RowVersion`
- `Tombstone`
- `LastOpSource`
- 工作区路径与快照版本信息

这种设计可以避免用户排序、筛选或调整 Excel 内容后导致记录与文件失去关联。

## 使用说明

### 初始化工作区

1. 启动 ExceLabel
2. 点击“选择目录”选择要管理的根目录
3. 点击“新建工作簿”或“已有工作簿”
4. 完成后系统会初始化工作区、扫描目录并生成 Excel 文件
5. 若监听已启动，后续保存工作簿可触发自动同步流程

### Excel -> 文件系统

1. 打开生成的工作簿
2. 仅修改 `A` 列 `NameStem`
3. 保存工作簿
4. 返回应用后执行“应用 Excel 修改”或保持监听器开启等待自动处理
5. 若存在非法名称或目标重名，会在 `Status` 列和界面冲突区中提示

### 文件系统 -> Excel

1. 在资源管理器或其他程序中修改、删除或新增根目录中的文件
2. 回到应用点击“同步文件状态”
3. 应用重新扫描目录并刷新 Excel 工作簿内容与状态

## 配置说明

当前版本主要通过 UI 输入和代码默认值完成配置，暂未提供独立配置文件。

### 运行期配置项

后端服务配置包含以下字段：

- `RootDir`：工作区根目录
- `WorkbookPath`：工作簿保存路径
- `AutoSyncEnabled`：是否启用自动监听
- `WorkbookDebounceMs`：工作簿保存监听去抖时间，默认 500ms
- `FsDebounceMs`：文件系统事件去抖时间，默认 800ms
- `MaxScanWorkers`：扫描并发数，默认 4

### 默认状态文件

工作区状态默认写入：

```text
<RootDir>/.excelabel/workspace-state.json
```

### 默认工作簿输出路径

如果使用根目录入口的默认配置启动，工作簿默认输出到：

```text
./output/excelabel.xlsx
```

实际桌面使用过程中，通常通过界面自行选择根目录与工作簿路径。

## 关键实现说明

### 文件身份识别

ExceLabel 不仅依赖文件路径，还结合以下信息识别同一文件：

- `RecordID`：首次导出时生成的稳定标识
- `Fingerprint`：由文件大小、修改时间、局部内容摘要组合得到

这样可以在文件发生改名或位置变化时尽量保留记录关联关系。

### 冲突控制

在应用 Excel 改名请求前，系统会校验：

- 文件名不能为空
- 文件名不能全为空白字符
- 文件名不能包含 Windows 非法字符 `\ / : * ? " < > |`
- 文件名不能以空格或点结尾
- 目标路径不能与同目录其他文件冲突

### 监听策略

由于 Excel 桌面应用并不直接向当前方案暴露单元格失焦事件，当前版本采用“保存后同步”的策略：

- 工作簿保存后触发 Excel -> 文件系统 差异分析
- 根目录文件变化后触发 文件系统 -> Excel 的重新对账

## 常见问题

### 1. 为什么应用提示工作簿被占用？

当应用需要回写状态列或重建工作簿时，如果 Excel 正在占用该文件，系统可能无法保存。此时请先关闭对应工作簿，再执行“同步文件状态”或重新应用状态写回。

### 2. 为什么 Excel 中有些列不可编辑？

当前设计只允许用户修改文件名本体，扩展名、相对目录、状态和隐藏元数据由系统维护，用于保证同步安全性。

### 3. 为什么改名后没有立即生效？

当前版本以“保存工作簿后触发”或手动点击按钮为主，不依赖 Excel 单元格级实时事件。

### 4. 为什么某些文件没有被扫描到？

以 `~$` 开头的临时文件会被主动忽略，以避免将 Excel 或 Office 临时锁文件导入清单。

### 5. 为什么出现“元数据异常”或“快照缺失”？

这通常表示工作簿隐藏元数据被破坏，或者当前 Excel 行无法与已有工作区快照对齐。建议重新执行一次工作区初始化或全量刷新。
